There are various ways you can create your layout system with hgrid. Use the flex or grid classes provided, or whip up something super fast and precise with the built-in, automatic content columns.

Containers

Keep your content from falling off to the sides by wrapping it in a .container-max. This lets you control the max-width of your page with the scss variable $container-max-width (or you can overwrite .container-max in your regular css).

Hgrid's 1440 pixel .container-max on a 1680 pixel screen.

The default .container-max limit is set to 1440 pixels. Use .container if you instead want to use the full viewport width.

In order to group content in vertical sections within a container, use the class .row (creates a flex container) or .row-block (creates a block container).

Creating layouts with the automatic content columns requires the use of .row. More on this below. .row-block is made for full width content that don't need a column, like headlines and text with floating images, etc.

.group can be used to section off multiple rows of content, like we have done on this page.

Automatic content columns

In order to quickly mark up a flexible layout grid, start by adding the .row class to an element. This creates a flex container with special margin settings for the column system to fit in. 

<div class="container-max">
<div class="row">
...
</div>
</div>

Any number of elements with the class .column inside the .row will divide its width equally: 

<div class="container-max">
<div class="row">
<div class="column">
<p>Column 1 (auto)</p>
</div>
<div class="column">
<p>Column 2 (auto)</p>
</div>
<div class="column">
<p>Column 3 (auto)</p>
</div>
</div>
</div>

The scss variable $column-gutter-width controls the distance between columns. You can also overwrite the regular CSS for .column and modify the default value of 0.8% on each side.

The columns will by default stack on viewports that are less than 769px wide. Breakpoints for responsiveness can be adjusted in _variables.scss. In the example below each row has a different number of columns.

Containers can be nested inside each other as long as you wrap each new level in a new .row. 

<div class="row">
<div class="column border-dark padding-5 radius-8"></div>
<div class="column border-dark padding-5 radius-8">
<div class="row margin-bottom-0">
<div class="column padding-10 bg-grey-light text-center">Bonjour</div>
<div class="column padding-10 bg-grey-light text-center">Bom dia</div>
</div>
</div>
</div>

As seen in this example, if we nest a .column-2 inside of another .column-2, the width of the inner column will be half of the outer one. Regardless of column size, for each level we nest, the parent element will act as the new "viewport" where its width holds a new context divided into 24 parts.

Accurate fractional columns

The .column-xx classes provide precise control over each column's span, with fractions based on the total container width. Remember to always wrap them in a .row.

hgrid divides the viewport in 24 fractional units of equal width. A full width column is called .column-1 and spans all 24 units. .column-2 spans 12 units, or 50% of the viewport width. .column-3 spans 33,33%, and so on up to .column-24. By combining fractional units we now get a precisely controlled layout grid: 

<div class="row">
<div class="column-2">
<p>.column-2</p>
</div>
<div class="column-4">
<p>.column-4</p>
</div>
<div class="column-4">
<p>.column-4</p>
</div>
</div>

This results in one column spanning 2/4 and two columns spanning 1/4, for a total of 4/4, or 100% of the parent container.
