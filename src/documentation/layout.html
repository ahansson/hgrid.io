There are a few different ways you can create a layout system with hgrid. Use the provided flex or grid utilities, or whip up something super fast and precise with the built-in, automatic content columns.

Containers

File location: sass/layout/_containers.scss (GitHub)

Keep your content from falling off to the sides by wrapping it in a .container-max. This lets you control the max-width of your page with the scss variable $container-max-width (or you can overwrite .container-max in your regular css).

The 1440 pixel .container-max on a 1680 pixel screen. The header uses the full-width .container.

The default .container-max width is set to 1440 pixels.

Use .container if you instead want to use the full viewport width.

In order to group content in vertical sections within a container, use the class .row (creates a flex container) or .row-block (creates a container for block elements).

Row

Creating layouts with the automatic content columns requires the use of .row. In the basic example below we have removed the default margin-bottom from paragraphs with the .mb-0 utility and center the text with .text-center: 

<div class="container">
  <div class="row">
    <div class="column">
      <p class="mb-0 text-center">Some content 1</p>
    </div>
    <div class="column">
      <p class="normal mb-0 text-center">Some other content 2</p>
    </div>
  </div>
  <div class="row">
    <div class="column">
      <p class="normal text-center mb-0">Some content 3</p>
    </div>
    <div class="column">
      <p class="text-center mb-0">Some other content 4</p>
    </div>
  </div>
</div>

As we can see, columns and rows have gutters/margin. Borders added for clarity.

You can use .row-block for full width content that don't need columns, such as headlines, article text with floating images, text columns, etc.

Column Stacking

.row stacks its columns vertically on screens less than 769px wide. This breakpoint is determined by the variable $stack-point that by default uses the width set in $phone-landscape. You can modify this in the @use rule when importing. You also have access to this breakpoint through the custom property --stack-point.

Custom Stacking Breakpoint

Having more than one breakpoint for stacking can be useful. Configure your alternative breakpoint in $custom-stack-point. Then, declare <div class="row custom-stack-point"> in your HTML.

Preventing Stacking

You can prevent columns from stacking altogether with .no-stack. HTML: <div class="row no-stack">.

Group

.group can be used to group rows of content into sections.
