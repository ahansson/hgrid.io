Learn how you can set up hgrid in your Node/NPM driven projects, and how to customize its default settings.

Vue

These recipes are tested with Vue v3.0.4.

Prerequisites

  1. Install two packages: npm i sass sass-loader --save-dev
  2. Install hgrid: npm install hgrid-css --save-dev
  3. Create main.scss, for example in assets/scss
  4. Initiate main.scss in your main.js file, respecting your local folder structure:
import './assets/scss/main.scss'
Method 1

Modify hgrid variables before the import

Configure hgrid default variables to your own likings before you import the full kit or individual parts.

This method is simple and flexible, as it injects hgrid into the global CSS namespace. The Sass maintainers are however planning to deprecate the @import rule sometime in the future and replace it entirely with @use and @forward which provide greater control by isolating classes and not polluting the global namespace. @import will continue to work in plain CSS. Note that CSS's @import is not the same as ES6' import.

// Override hgrid default variables
$container-max-width: 1333px;
$font-color: #333;
$link-color: #ffe88d;
$link-color-hover: #fff0b4;

// Import hgrid from node_modules
@import '~hgrid-css/sass/hgrid';

// Import your own sass or regular css partials, if you have any
@import 'partials/_app-frame.scss';
@import 'partials/_theme.scss';

// Add your own css/scss from here down, for example:
:root {
  --some-color: #ffe88d;
  --some-other-variable: 2px solid #999;
  --huge-title-size: 3.2rem;
}

.huge-title {
  font-size: var(--huge-title-size);
  &:hover {
    color: #ff5400;
  }
}

// ... etc.

You could also make your bundle lighter by importing only certain parts:

@import '~hgrid-css/sass/variables/variables'; // Required
@import '~hgrid-css/sass/base/base'; // Required
@import '~hgrid-css/sass/mixins/mixins'; // Required

@import '~hgrid-css/sass/layout/flex';
@import '~hgrid-css/sass/layout/containers';
@import '~hgrid-css/sass/layout/helpers/adjustment';

Regular CSS variables like --sample-var: #111; from hgrid can be overridden in your own :root { ... } class. Regular CSS variables from hgrid are available to be used anywhere in your project, including in components, thanks to @import (as opposed to @use which would isolate them to the scope where they are "@used").

$sass-variables from hgrid do not work in component files unless you import sass/variables/variables.scss at the top of each file. Or, you can get access to them via vue.config.js like explained here. It's recommended to keep hgrid in the global context and use these variables in the global stylesheets outside of your .vue files.

Method 2

Include the full kit with @use

In your main.scss, import hgrid as shown below. Modify hgrid variables you want to change. Note that we use commas after the variable declarations, not semicolons:

@use 'hgrid-css/sass/hgrid' with (
  $link-color: #ffe88d,
  $link-color-hover: #fff0b4,
  $container-max-width: 1600px,
  $link-underline-color: #ffe88d
);

// Import your own scss or regular css partials, if you have any
@import 'partials/_app-frame.scss';
@import 'partials/_theme.scss';

// Other styles 
.alertbox {
  ...
}

// etc.

hgrid will now be fully loaded into your project with your own variable overrides, if you added any.

You can’t use hgrid's variables once @use has been executed by the compiler. The @use rule compiles to an isolated CSS namespace. It requires to be included at the top of your files. Trying to @import anything else before the @use rule will not compile.

If you want to leave certain partials from hgrid out, you have to use one @use rule for each partial to include.

Read more about the @use rule over at sass-lang.com.

Method 3

Quick imports


3a. Import in 'App.vue'

Import either Sass or precompiled CSS at the top of App.vue's <style> tag.

CSS
<style lang="css">
  @import '~hgrid-css/dist/hgrid.min.css';

  /* Other CSS classes from here down */
</style>
SCSS
<style lang="scss">

  // Override hgrid defaults
  $container-max-width: 1400px;
  $font-color: #888;

  // Import everything from hgrid
  @import '~hgrid-css/sass/hgrid';

  // ... or just the parts you need
  @import '~hgrid-css/sass/variables/variables'; // Required
  @import '~hgrid-css/sass/base/base'; // Required
  @import '~hgrid-css/sass/mixins/mixins'; // Required
  @import '~hgrid-css/sass/layout/flex';
  @import '~hgrid-css/sass/layout/containers';

  // Add your own css/scss from here down, for example:
  @import 'partials/_app-frame.scss';

  :root {
    --type-size: 1.88rem;
  }

  .body {
    font-size: var(--type-size);
  }

  // ... etc.
</style>

Please refer to prerequisites for setting up Sass in Vue.

Remember that scoped styles will be loaded before hgrid when Vue stitches together its virtual DOM. This means hgrid may interfere with certain styles even though they are scoped to a component. This can be avoided with @use from method 2.


3b. Import in 'main.js'
import ‘~hgrid-css/dist/hgrid.min.css’

3c. Import prebuilt CSS to local CSS

At the top of a local stylesheet in for example public/assets/ you can import the full hgrid kit like this:

@import '~hgrid-css/dist/hgrid.min.css;

Global access to Sass variables with vue.config.js

Another way to access to the hgrid $variables from within components is to make them available from the config like so:

module.exports = {
   css: {
     loaderOptions: {
       scss: {
         additionalData: `
           @import '~hgrid-css/sass/variables/_variables.scss';
         `
      }
    }
  }
}

⚠️ Important: Do not import anything here that won't be abstracted away in the build process. If you do, performance will be affected. The same styles will be included multiple times in the bundle – once for every component and view.

Restart the dev server with npm run serve for config changes to take effect.

React

This recipe is verified with React 17.0.1 and create-react-app 4.0.2.

Demo project

Check out the starter project on GitHub.

Prerequisites

  1. Install the Dart Sass package: npm i sass --save-dev
  2. Install hgrid: npm i hgrid-css --save-dev
  3. Rename index.css to index.scss (or create it in src if it doesn't exist.
  4. Rename App.css and other component files to use the file ending .scss.
  5. Import your .scss files in index.js and the relevant component files:
import './index.scss';
import './App.scss';

Any files with the .scss extension imported in App.js will be automatically compiled to CSS. Autoprefixer is a part of the hgrid project for broader browser compatibility, but in a React app from scratch you have to configure it yourself. Create-react-app and the starter project we link to above has Autoprefixer working out of the box.

@import to index.scss

@charset "utf-8";

// Override hgrid default variables if you wish
$container-max-width: 1333px;
$font-color: #333;

// Import everything from hgrid
@import '~hgrid-css/sass/hgrid';

// ... or just the parts you need
@import '~hgrid-css/sass/variables/variables'; // Required
@import '~hgrid-css/sass/base/base'; // Required
@import '~hgrid-css/sass/mixins/mixins'; // Required
@import '~hgrid-css/sass/layout/flex';
@import '~hgrid-css/sass/layout/containers';
@import '~hgrid-css/sass/layout/helpers/adjustment';

// Add your own scss/css below
@import 'partials/_app-frame.scss';

body {
  // etc.
}

You now have access to hgrid's utilities from any component file in your app.

import './App.scss';

function App() {
  return (
    <div className="App container-max">
      <div className="row padding-top-150">
        <div className="column radius-6 bg-gray-light">Content</div>
        <div className="column radius-6 bg-dark text-white">Content here</div>
        <div className="column radius-6 bg-gray-darker text-white">More content</div>
        <div className="column radius-6 bg-white text-blue">Hello content</div>
      </div>
    </div>
  );
}

export default App;

If you need access to specifics from hgrid in your components, you can import partials in the component scss.

@import '~hgrid-css/sass/variables/variables.scss';

.ExcellentComponent {
  font-size: 20px;
  .column {
    padding: 1rem 2rem;
  }

  // See if a hgrid default variable is available:
  @media (max-width: $phone-landscape) {
    text-align: center;
  }
}

// Here we override a hgrid class which has a customized value from 'index.scss' 
// and set it back to the original variable value we imported above

.container-max {
  max-width: $container-max-width;
}

Inspect the page and you will see that the 1200px $container-max-width variable we configured in index.scss has been overridden by the default variable's value of 1440px that we imported into this file.

Overrides in index.scss will display in dev tools as coming from hgrid's own partial files. This is correct, as variables you customize get injected during compilation.

Our React starter project which is based on create-react-app automatically minifies and autoprefixes the CSS output. Read more in the docs.
